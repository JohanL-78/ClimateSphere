🎯 OBJECTIF DE L'APPLICATION
============================
Climate Sphere est une application web moderne qui visualise les données d'anomalies 
de température NASA GISTEMP à travers :
- Un globe 3D interactif avec Three.js
- Des tableaux de données dynamiques
- Des graphiques temporels avec Recharts
- Une interface responsive optimisée

📁 ARCHITECTURE GÉNÉRALE
========================

TECHNOLOGIE DE BASE : Next.js 15.3.3 (App Router)
├── Framework UI : React 19.0.0
├── Visualisation 3D : Three.js + @react-three/fiber
├── Animations : Framer Motion  
├── Graphiques : Recharts
├── Styling : CSS-in-JS inline + CSS modules
└── Performance : Bundle splitting + Lazy loading

🔄 FLUX DE DONNÉES ET INTERACTIONS
==================================

1. CHARGEMENT INITIAL
   app/page.js 
   → Server Action (lib/actions.js) 
   → Récupération CSV NASA GISTEMP
   → Passage props à Dashboard

2. ARCHITECTURE DES COMPOSANTS
   Dashboard.jsx (État global)
   ├── Navbar.jsx (Navigation)
   ├── Sidebar.jsx (Onglets + GlobeControls)
   │   └── GlobeControls.jsx (Année/Mois + DataCards)
   ├── MainContent.jsx (Rendu conditionnel par onglet)
   │   ├── LazyClimateGlobe.jsx → ClimateGlobe.jsx
   │   ├── DataTable.jsx 
   │   └── LazyTemperatureChart.jsx → TemperatureChart.jsx
   └── DetailModal.jsx (Modal données température)

3. COMMUNICATION INTER-COMPOSANTS
   
   Dashboard → Sidebar → GlobeControls
   ↓ controls, onControlsChange
   
   Dashboard → MainContent → LazyClimateGlobe → ClimateGlobe
   ↓ availableDates, controls, onControlsChange
   
   ClimateGlobe → (onLoad) → Dashboard
   ↑ Retour données température pour sidebar

🧩 POINTS TECHNIQUES ESSENTIELS
===============================

A) LAZY LOADING ET PERFORMANCE
-------------------------------
• LazyClimateGlobe.jsx : Dynamic import pour Three.js (chunk séparé)
• LazyTemperatureChart.jsx : Dynamic import pour Recharts  
• Bundle splitting dans next.config.js pour optimiser le chargement
• Réduction géométrie 3D : 128x64 → 64x32 triangles (-75% polygones)

B) GESTION DES ÉTATS
--------------------
Dashboard.jsx centralise 3 états principaux :
• sidebarOpen : Affichage sidebar
• activeTab : Onglet actif (globe/data/chart/about)  
• globeControls : État complexe du globe
  {
    year, month,           // Période sélectionnée
    temps: {},             // Données température chargées
    isLoadingTemps,        // État chargement
    modalOpen, modalType   // Modal de détails
  }

C) RESPONSIVE DESIGN
--------------------
useResponsive.js hook centralisé :
• isMobile, isTablet, isDesktop
• Debounce sur resize pour performance
• Utilisé dans tous les composants pour adaptation UI

D) OPTIMISATIONS LIGHTHOUSE
---------------------------
• Préconnexions DNS pour fonts Google
• Cache headers pour assets statiques (textures)
• Animations GPU avec willChange + transform
• Gradient CSS vs image 884kB (économie 99.9%)
• Font optimization avec next/font/google
